# ROADMAP EVALUATION TAM - APLIKASI TMS NURANI
## Technology Acceptance Model untuk Teaching Management System MTS Nurul Aiman

---

## üéØ MODEL TAM UNTUK TMS NURANI

### **EXTERNAL VARIABLES (Variabel Eksternal)**

#### 1. System Characteristics (Karakteristik Sistem)
- User Interface Design (Desain antarmuka yang intuitif)
- System Functionality (Kelengkapan fitur)
- System Reliability (Keandalan sistem)
- Response Time (Kecepatan akses)
- Mobile Responsiveness (Akses dari berbagai device)

#### 2. User Characteristics (Karakteristik Pengguna)
- Computer Self-Efficacy (Kemampuan teknologi guru/TU)
- Prior Experience (Pengalaman sistem digital sebelumnya)
- Age & Education Level (Usia dan tingkat pendidikan)
- Technology Anxiety (Kecemasan terhadap teknologi)

#### 3. Organizational Factors (Faktor Organisasi)
- Management Support (Dukungan Kepala Sekolah)
- Training Quality (Kualitas pelatihan)
- Technical Support Availability (Ketersediaan bantuan teknis)
- Peer Influence (Pengaruh rekan kerja)

#### 4. Task Characteristics (Karakteristik Tugas)
- Task Complexity (Kompleksitas tugas mengajar)
- Task Relevance (Relevansi sistem dengan pekerjaan)
- Job Fit (Kesesuaian dengan kebutuhan kerja)

---

### **PERCEIVED USEFULNESS (PU) - Persepsi Kegunaan**

#### Indikator untuk TMS Nurani:
1. **Efisiensi Kerja**
   - Mengurangi waktu administrasi guru
   - Mempercepat input data presensi
   - Memudahkan pembuatan laporan

2. **Produktivitas**
   - Meningkatkan fokus pada pembelajaran
   - Mengurangi pekerjaan manual
   - Otomasi proses rutin

3. **Efektivitas**
   - Akurasi data lebih tinggi
   - Akses informasi real-time
   - Integrasi data terpusat

4. **Manfaat Spesifik TMS**
   - Manajemen materi pembelajaran lebih terorganisir
   - Tracking pertemuan otomatis
   - Presensi digital guru & siswa
   - Export laporan mudah
   - Kalender & pengumuman terintegrasi

#### Pertanyaan Evaluasi PU:
- Apakah TMS meningkatkan efisiensi pekerjaan saya?
- Apakah TMS memudahkan pengelolaan data mengajar?
- Apakah TMS bermanfaat untuk tugas sehari-hari?
- Apakah TMS meningkatkan produktivitas saya?
- Apakah TMS membantu saya bekerja lebih cepat?

---

### **PERCEIVED EASE OF USE (PEOU) - Persepsi Kemudahan**

#### Indikator untuk TMS Nurani:
1. **Kemudahan Belajar**
   - Mudah dipahami tanpa training intensif
   - Kurva pembelajaran singkat
   - Intuitif untuk pengguna baru

2. **Kemudahan Penggunaan**
   - Navigasi jelas dan sederhana
   - Fitur mudah ditemukan
   - Proses input data tidak rumit

3. **Kemudahan Interaksi**
   - Interface user-friendly
   - Instruksi jelas
   - Feedback sistem informatif

4. **Fitur Spesifik TMS yang Mudah**
   - Login sederhana
   - Dashboard informatif
   - Upload file drag-and-drop
   - Import Excel otomatis
   - Clock in/out presensi 1-klik
   - Filter & search data mudah

#### Pertanyaan Evaluasi PEOU:
- Apakah TMS mudah dipelajari?
- Apakah navigasi TMS jelas dan mudah dipahami?
- Apakah saya mudah menjadi terampil menggunakan TMS?
- Apakah interaksi dengan TMS mudah dimengerti?
- Apakah TMS tidak memerlukan banyak usaha mental?

---

### **ATTITUDE TOWARD USING (ATU) - Sikap Terhadap Penggunaan**

#### Indikator untuk TMS Nurani:
1. **Evaluasi Positif/Negatif**
   - Menggunakan TMS adalah ide yang baik
   - TMS membuat pekerjaan lebih menyenangkan
   - Senang bekerja dengan TMS

2. **Kepuasan Pengguna**
   - Puas dengan fitur yang tersedia
   - Nyaman menggunakan sistem
   - Lebih suka TMS daripada cara manual

3. **Kepercayaan Sistem**
   - Percaya data aman
   - Yakin sistem reliable
   - Percaya sistem membantu pekerjaan

#### Pertanyaan Evaluasi ATU:
- Apakah menggunakan TMS adalah ide yang baik?
- Apakah saya senang menggunakan TMS?
- Apakah bekerja dengan TMS menyenangkan?
- Apakah saya merasa nyaman menggunakan TMS?
- Apakah saya puas dengan TMS?

---

### **BEHAVIORAL INTENTION TO USE (BI) - Niat Perilaku Menggunakan**

#### Indikator untuk TMS Nurani:
1. **Niat Berkelanjutan**
   - Berencana terus menggunakan TMS
   - Akan menggunakan TMS secara rutin
   - Tidak akan kembali ke cara manual

2. **Rekomendasi**
   - Akan merekomendasikan ke rekan guru
   - Akan mendorong orang lain menggunakan
   - Akan berbagi pengalaman positif

3. **Eksplorasi Fitur**
   - Berencana menggunakan lebih banyak fitur
   - Tertarik mempelajari fitur lanjutan
   - Akan memanfaatkan semua fungsi TMS

#### Pertanyaan Evaluasi BI:
- Apakah saya berniat terus menggunakan TMS?
- Apakah saya akan merekomendasikan TMS ke rekan kerja?
- Apakah saya berencana menggunakan TMS secara rutin?
- Apakah saya akan menggunakan lebih banyak fitur TMS?
- Apakah saya berkomitmen menggunakan TMS jangka panjang?

---

### **ACTUAL SYSTEM USE (ASU) - Penggunaan Sistem Aktual**

#### Indikator untuk TMS Nurani:
1. **Frekuensi Penggunaan**
   - Jumlah login per minggu
   - Durasi sesi penggunaan
   - Konsistensi penggunaan harian

2. **Intensitas Penggunaan**
   - Jumlah fitur yang digunakan
   - Kedalaman penggunaan fitur
   - Variasi aktivitas dalam sistem

3. **Pola Penggunaan Spesifik TMS**
   - **Guru:**
     - Frekuensi input materi pembelajaran
     - Frekuensi presensi (clock in/out)
     - Frekuensi input presensi siswa
     - Frekuensi tracking pertemuan
     - Frekuensi akses jadwal
   
   - **TU:**
     - Frekuensi input data siswa/guru
     - Frekuensi import Excel
     - Frekuensi export laporan
     - Frekuensi update kalender/pengumuman
   
   - **Kepala Sekolah:**
     - Frekuensi monitoring dashboard
     - Frekuensi review laporan
     - Frekuensi approval surat

4. **Kelengkapan Data**
   - Persentase data yang terisi lengkap
   - Kualitas data yang diinput
   - Update data secara berkala

#### Metrik Evaluasi ASU:
- Login rate: % pengguna yang login minimal 2x/minggu
- Feature adoption rate: % fitur yang digunakan
- Data completeness: % kelengkapan data
- Task completion rate: % tugas yang diselesaikan via TMS
- Time spent: Rata-rata durasi penggunaan per hari

**H1:** External Variables berpengaruh positif terhadap Perceived Usefulness

**H7:** Attitude Toward Using berpengaruh positif terhadap Behavioral Intention
- Sikap positif ‚Üí Niat menggunakan tinggi

**H8:** Behavioral Intention berpengaruh positif terhadap Actual System Use
- Niat tinggi ‚Üí Penggunaan aktual tinggi

## üóìÔ∏è TIMELINE EVALUASI TAM

### **FASE 1: PRE-IMPLEMENTATION (Minggu -2)**
**Tujuan:** Baseline measurement

**Aktivitas:**
- Ukur External Variables (karakteristik user, organisasi)
- Pre-test ekspektasi PU & PEOU
- Identifikasi concerns & barriers
- Dokumentasi proses manual existing

**Output:**
- Baseline data
- User profiling
- Ekspektasi awal

---

### **FASE 2: EARLY ADOPTION (Minggu 1-2)**
**Tujuan:** Initial TAM assessment

**Aktivitas:**
- Daily monitoring ASU (login rate, feature usage)
- Quick survey PEOU & PU (mini TAM)
- Observasi kemudahan belajar sistem
- Kumpulkan feedback awal

**Metrik TAM:**
- PEOU: Kemudahan di hari-hari awal
- PU: Persepsi manfaat awal
- ASU: Tingkat penggunaan aktual

**Output:**
- Early adoption report
- Quick wins & pain points
- Initial TAM scores

---

### **FASE 3: MID-TERM EVALUATION (Minggu 3-4)**
**Tujuan:** Comprehensive TAM measurement

**Aktivitas:**
- **Kuesioner TAM Lengkap** (semua konstruk)
- Wawancara mendalam (PU, PEOU, ATU, BI)
- Focus Group Discussion per user group
- Analisis korelasi antar konstruk TAM
- Validasi hipotesis TAM

**Metrik TAM (SEMUA):**
- PU score & distribusi
- PEOU score & distribusi
- ATU score & distribusi
- BI score & distribusi
- ASU metrics (login, feature adoption)

**Analisis Statistik:**
- Correlation analysis (PEOU ‚Üî PU, PU ‚Üî ATU, dll)
- Regression analysis (pengaruh PEOU & PU terhadap ATU)
- Path analysis (model TAM lengkap)
- Gap analysis (ekspektasi vs realitas)

**Output:**
- Comprehensive TAM report
- Validated TAM model untuk TMS
- Identifikasi faktor kunci penerimaan
- Rekomendasi improvement

---

### **FASE 4: LONG-TERM SUSTAINABILITY (Bulan 2-3)**
**Tujuan:** Sustained usage & improvement

**Aktivitas:**
- Monitoring kontinyu ASU
- Survey kepuasan bulanan (simplified TAM)
- Evaluasi perubahan ATU & BI over time
- Implementasi improvements berdasarkan TAM findings

**Metrik TAM:**
- BI: Niat berkelanjutan
- ASU: Konsistensi penggunaan
- ATU: Perubahan sikap dari bulan 1

**Output:**
- Trend analysis
- Sustainability report
- Long-term improvement plan

---

### **FASE 5: SEMESTER EVALUATION (Bulan 6)**
**Tujuan:** Long-term impact assessment

**Aktivitas:**
- Kuesioner TAM semester (perbandingan dengan bulan 1)
- Evaluasi ROI & dampak sistem
- Analisis sustained adoption
- Success stories documentation

**Metrik TAM:**
- Perbandingan semua konstruk: Bulan 1 vs Bulan 6
- Long-term adoption rate
- System maturity level

**Output:**
- Semester TAM report
- Success factors identification
- Future enhancement roadmap

## üìä METODE ANALISIS DATA TAM

### **1. Analisis Deskriptif**
- Mean, median, standard deviation untuk setiap konstruk
- Distribusi frekuensi
- Persentase kategori (sangat setuju, setuju, dll)

### **2. Analisis Reliabilitas**
- Cronbach's Alpha untuk setiap konstruk (target ‚â• 0.7)
- Item-total correlation

### **3. Analisis Validitas**
- Content validity (expert judgment)
- Construct validity (factor analysis)
- Convergent & discriminant validity

### **4. Analisis Korelasi**
- Pearson correlation antar konstruk TAM
- Validasi hubungan teoritis

### **5. Analisis Regresi**
- Multiple regression: PEOU & PU ‚Üí ATU
- Simple regression: ATU ‚Üí BI
- Simple regression: BI ‚Üí ASU

### **6. Analisis Jalur (Path Analysis)**
- Uji model TAM lengkap
- Direct & indirect effects
- Model fit indices

### **7. Analisis Komparatif**
- Perbandingan antar user groups (Guru vs TU vs Kepsek)
- Perbandingan antar periode waktu (Minggu 1 vs Minggu 4 vs Bulan 3)
- Pre-post comparison (ekspektasi vs realitas)

## üöÄ ACTION PLAN BERDASARKAN HASIL TAM

### **Jika PEOU Rendah (<3.0):**
**Masalah:** Sistem sulit digunakan

**Action:**
- Perbaiki UI/UX yang membingungkan
- Tambahkan tutorial & tooltips
- Simplifikasi workflow
- Training tambahan
- Quick reference guide

---

### **Jika PU Rendah (<3.0):**
**Masalah:** Sistem tidak terlihat bermanfaat

**Action:**
- Komunikasikan value proposition lebih jelas
- Showcase success stories & time savings
- Demonstrasi ROI
- Highlight fitur yang paling membantu
- Kumpulkan & address specific concerns

---

### **Jika ATU Negatif (<3.0):**
**Masalah:** Sikap negatif terhadap sistem

**Action:**
- Change management workshops
- Address resistance to change
- Involve users dalam improvement
- Peer mentoring program
- Management support & endorsement

---

### **Jika BI Rendah (<3.0):**
**Masalah:** Tidak berniat terus menggunakan

**Action:**
- Identifikasi barriers to continued use
- Improve PU & PEOU (root causes)
- Incentive program untuk active users
- Make system mandatory (dengan persetujuan)
- Regular communication tentang benefits

---

### **Jika ASU Rendah (Login rate <60%):**
**Masalah:** Penggunaan aktual rendah

**Action:**
- Investigate gap antara BI dan ASU
- Remove technical barriers
- Improve system reliability
- Provide better support
- Monitor & remind inactive users
